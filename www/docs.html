API Documentation

The ipns.app API allows you to create and manage IPNS records programmatically.

You will need to generate an API key to gain access to the API.

Your API Key should be added as an "Authorization" header with the value "Bearer YOUR_API_KEY"

The following endpoints are currently enabled. Endpoints are grouped by their allowed permissions.

"READ", "SELF ONLY"*

- GET https://ipns.app/api/record/list
- GET https://ipns.app/api/record/:id

"WRITE", "SELF ONLY"*

- POST https://ipns.app/api/record
{"alias": "NICKNAME_FOR_IPNS_ADDRESS"}

- PUT https://ipns.app/api/record/:id
{"cid": "IPFS_CID"} || {"json": "JSON_STRING"}

- DELETE https://ipns.app/api/record/:id

* API Keys with the "SELF ONLY" permission have READ + WRITE permissions, but only for IPNS Records that they themselves create.
